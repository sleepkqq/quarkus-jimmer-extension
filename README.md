# Quarkus Jimmer Extension

[![CI](https://github.com/sleepkqq/quarkus-jimmer-extension/actions/workflows/ci.yml/badge.svg)](https://github.com/sleepkqq/quarkus-jimmer-extension/actions/workflows/ci.yml)
[![Release](https://github.com/sleepkqq/quarkus-jimmer-extension/actions/workflows/release.yml/badge.svg)](https://github.com/sleepkqq/quarkus-jimmer-extension/actions/workflows/release.yml)
[![JitPack](https://jitpack.io/v/sleepkqq/quarkus-jimmer-extension.svg)](https://jitpack.io/#sleepkqq/quarkus-jimmer-extension)
[![License](https://img.shields.io/badge/license-Apache%202.0-blue.svg)](LICENSE)

A [Quarkus](https://quarkus.io) extension for the [Jimmer](https://github.com/babyfish-ct/jimmer)
ORM. It wires Jimmer into Quarkus' CDI container, generates repository implementations at build
time, supports multiple datasources, and — the focus of this fork — makes the whole stack work in
**GraalVM native image**. Java **and** Kotlin are both first-class.

Based on [flynndi/quarkus-jimmer-extension](https://github.com/flynndi/quarkus-jimmer-extension).

## Why this fork

The upstream extension targets JVM mode only. Jimmer leans heavily on reflection and on classes
generated by its annotation processor, so a naive native build fails with
`ClassNotFoundException` / missing-reflection errors at runtime. This fork adds the build-time
configuration that makes it work:

- **GraalVM native support** — Jimmer SQL annotations, generated `Draft`/`Props`/`Fetcher` classes
  and the runtime types the engine touches are registered for native reflection (see
  `deployment` + `reflect-config.json`). `./gradlew build -Dquarkus.native.enabled=true` produces a
  working native binary.
- **Build-time repository codegen** — interfaces extending `JRepository` / `KRepository` are
  discovered via Jandex and their implementations are generated as bytecode at build time — no
  reflective proxies at runtime.
- **CDI-native wiring** — `JSqlClient` / `KSqlClient` and every repository are synthetic CDI beans.
- **Multi-datasource** — each datasource gets its own `JSqlClient` bean qualified with
  `@DataSource("name")`.
- **Connection management** — one JDBC connection per JTA transaction, delegated to Agroal +
  Narayana.

## Configuration

Configure the extension under `quarkus.jimmer.*`. Minimal `application.yml`:

```yaml
quarkus:
  jimmer:
    language: java            # or: kotlin
    dialect: org.babyfish.jimmer.sql.dialect.PostgresDialect
    show-sql: true
    pretty-sql: true
  datasource:
    db-kind: postgresql
    jdbc:
      url: jdbc:postgresql://localhost:5432/music
```

Common keys (per-datasource keys accept a datasource name segment, `<default>` for the main one):

| Key | Default | Purpose |
|---|---|---|
| `quarkus.jimmer.language` | `java` | Codegen flavour: `java` or `kotlin` |
| `quarkus.jimmer.<ds>.show-sql` | `false` | Log executed SQL |
| `quarkus.jimmer.<ds>.pretty-sql` | `false` | Pretty-print logged SQL |
| `quarkus.jimmer.<ds>.dialect` | auto-detected | Jimmer SQL dialect |
| `quarkus.jimmer.<ds>.default-schema` | — | Default schema |
| `quarkus.jimmer.<ds>.trigger-type` | `BINLOG_ONLY` | Trigger strategy |
| `quarkus.jimmer.database-validation.mode` | `NONE` | Validate schema against entities on start |

When your entities live in a **library jar** (not the application module), point Quarkus' indexer
at it so the build step can discover them:

```yaml
quarkus:
  index-dependency:
    catalog:
      group-id: com.example
      artifact-id: music-catalog-model
```

## Usage

Define your entities the usual Jimmer way, then declare a repository interface — its implementation
is generated at build time and exposed as a CDI bean.

### Entities

A small shared base type plus a few associated entities:

```java
@MappedSuperclass
public interface BaseEntity {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    long id();

    @Version
    int version();
}

@Entity
public interface Artist extends BaseEntity {

    @Key
    String name();

    String country();

    @OneToMany(mappedBy = "artist")
    List<Album> albums();
}

@Entity
public interface Album extends BaseEntity {

    @Key
    String title();

    LocalDate releasedOn();

    @ManyToOne
    @OnDissociate(DissociateAction.DELETE)
    Artist artist();

    @ManyToMany
    @JoinTable(
            name = "album_genre",
            joinColumnName = "album_id",
            inverseJoinColumnName = "genre_id"
    )
    List<Genre> genres();

    @OneToMany(mappedBy = "album")
    List<Track> tracks();
}

@Entity
public interface Track extends BaseEntity {

    String title();

    int durationSeconds();

    @ManyToOne
    @OnDissociate(DissociateAction.DELETE)
    Album album();
}
```

### Java

```java
// Repository — implementation generated at build time.
// Derived methods are parsed from the method name.
public interface AlbumRepository extends JRepository<Album, Long> {

    List<Album> findByArtistName(String name);

    boolean existsByTitle(String title);

    // Custom typed query through the injected SqlClient
    default List<Album> findReleasedAfter(LocalDate date) {
        AlbumTable album = AlbumTable.$;
        return sql()
                .createQuery(album)
                .where(album.releasedOn().gt(date))
                .orderBy(album.releasedOn().desc())
                .select(album)
                .execute();
    }
}

@ApplicationScoped
public class AlbumService {

    @Inject
    AlbumRepository albumRepository;

    @Transactional
    public Album release(Artist artist, String title) {
        return albumRepository.save(
                AlbumDraft.$.produce(draft -> {
                    draft.setTitle(title);
                    draft.setReleasedOn(LocalDate.now());
                    draft.setArtist(artist);
                }));
    }

    @Transactional(TxType.SUPPORTS)
    public Album findById(long id) {
        return albumRepository.findNullable(id);
    }
}

// Need the raw client somewhere? Inject it directly.
@Inject
JSqlClient sql;
```

### Kotlin

```kotlin
// Repository
interface AlbumRepository : KRepository<Album, Long> {

    fun findByArtistName(name: String): List<Album>

    fun existsByTitle(title: String): Boolean

    // Custom typed query — `sql` is available inside default methods
    fun findReleasedAfter(date: LocalDate): List<Album> =
        sql.createQuery(Album::class) {
            where(table.releasedOn gt date)
            orderBy(table.releasedOn.desc())
            select(table)
        }.execute()
}

@ApplicationScoped
class AlbumService(
    private val albumRepository: AlbumRepository,
) {

    @Transactional
    fun release(artist: Artist, title: String): Album =
        albumRepository.save(
            Album {
                this.title = title
                this.releasedOn = LocalDate.now()
                this.artist = artist
            })

    @Transactional(Transactional.TxType.SUPPORTS)
    fun findById(id: Long): Album? = albumRepository.findNullable(id)
}

// Inject the raw client directly when you need it
@Inject
lateinit var sql: KSqlClient
```

### DTOs / Views (optional)

Jimmer's `.dto` files (in `src/main/dto/`) generate type-safe projections; query for them
directly:

```
export com.example.catalog.Album

AlbumSummaryView {
    title
    releasedOn
    id(artist) as artistId
    genres { name }
}
```

```kotlin
fun listSummaries(): List<AlbumSummaryView> =
    sql.createQuery(Album::class) {
        orderBy(table.releasedOn.desc())
        select(table.fetch(AlbumSummaryView::class))
    }.execute()
```

### Customizing the SqlClient

Register a `Customizer` / `KCustomizer` CDI bean to tweak the builder (e.g. add a
`DraftInterceptor` that stamps audit fields):

```java
@ApplicationScoped
public class CatalogCustomizer implements Customizer {

    @Override
    public void customize(JSqlClient.Builder builder) {
        builder.addDraftInterceptor(new AuditInterceptor());
    }
}
```

For more on entities, fetchers, save commands and the query DSL, see the
[Jimmer documentation](https://babyfish-ct.github.io/jimmer-doc/).

## Native image

```bash
./gradlew build -Dquarkus.native.enabled=true
# or a container build, no local GraalVM required:
./gradlew build -Dquarkus.native.enabled=true -Dquarkus.native.container-build=true
```

The extension registers the reflection metadata Jimmer needs, so generated entities, drafts,
fetchers and repositories work without extra `@RegisterForReflection` annotations in your app.

## Build

```bash
./gradlew build                          # full build + tests
./gradlew :quarkus-jimmer:compileJava    # compile runtime only
./gradlew publishToMavenLocal            # publish to local Maven
./gradlew integration-tests:test         # integration tests (Docker required: Postgres + Redis)
./gradlew syncReadme                     # sync README versions from libs.versions.toml
```

JDK 21, Gradle 9.4.0.
